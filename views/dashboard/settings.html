Zusammenspiel mit Moodle

      
Das Chat-Widget wird über ein TinyMCE-Snippet in eine Moodle-Aktivität eingebettet:

      
    
        
  1. Lehrkraft öffnet eine Moodle-Textseite oder Aufgabe im Bearbeitungsmodus.
    2. 
        
  2. Im TinyMCE-Editor: Einfügen → Snippets → Snippet abgpt oder tegpt auswählen.
    3. 
        
  3. Das Snippet bettet ein <iframe> ein, das chat.html lädt und die Aktivitäts-ID übergibt.
    4. 
        
  4. Schüler sehen beim Öffnen der Aktivität das schwebende Chat-Widget.
    5. 
        
  5. Nachrichten laufen per WebSocket zum Server — Lehrkraft sieht alles live im Chats-Tab.
    6. 
      

      
Jede Moodle-Aktivität hat eine eigene Aktivitäts-ID, eigenen System-Prompt und eigene Kosten-Statistik.

    


    

      
Tipps & FAQ

      

        
Wie ändere ich den System-Prompt?

        
Tab Einstellungen → Abschnitt „Systemprompt & Globalmodell". Änderungen gelten sofort für neue Chats.


        
Was sind Personas?

        
Virtuelle Schüler-Profile, mit denen der System-Prompt im Chats-Tab (Bereich Simulation) getestet werden kann.


        
Was bedeuten die Token-Kosten?

        
Jede KI-Antwort verbraucht Token. Die Kostenanzeige im Chats-Tab summiert alle Kosten der Aktivität in US-Dollar.


        
Kann ich den Chat für Schüler sperren?

        
Ja: Im Chats-Tab gibt es einen Plenum-Modus. Wenn aktiv, wird das Widget für Schüler deaktiviert.

Chat-Nachrichtenfluss

      
sequenceDiagram
    participant S as Schüler-Browser
    participant W as Widget (moo-bot.js)
    participant SV as Server (server.js)
    participant OAI as OpenAI API
    participant D as Dashboard – Chats

    S->>W: Nachricht eingeben + senden
    W->>SV: WebSocket: { type: "message", text }
    SV->>SV: Nachricht in DB speichern
    SV->>OAI: Responses API (stream: true)
    loop Streaming
        OAI-->>SV: Text-Chunk
        SV-->>W: WebSocket: { type: "chunk", text }
        W-->>S: Chunk im Chat anzeigen
    end
    OAI-->>SV: done + Token-Anzahl
    SV->>SV: Token-Log in DB speichern
    SV-->>W: WebSocket: { type: "done" }
    SV-->>D: WebSocket broadcast { type: "message", ... }
