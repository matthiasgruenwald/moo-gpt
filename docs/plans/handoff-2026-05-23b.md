## Was diese Session erledigt hat

### Issue #58 — Dashboard-Umbau ✅

`chats.html` hat jetzt nur noch **Schüler** und **Optimierung** im tab-bar. Einstellungen, Info und Admin sind auf eine eigene Seite ausgelagert:

| Datei | Änderung |
|-------|----------|
| `views/dashboard/chats.html` | 3 Tabs + 3 Panel-HTML-Blöcke + panel-CSS entfernt |
| `views/dashboard/settings.html` | Vollständig neu: settings / info / admin Panel mit Tab-Bar |
| `public/settings.js` | Neu — alle Settings-Funktionen eigenständig (kein dashboard.js) |
| `public/dashboard.js` | 8 Null-Guards: `?.classList`, `?.addEventListener`, `loadSettings`-Guard |

**Nicht-offensichtlich:** `settings.js` darf `loadPersonas()` nicht aufrufen (die Funktion existiert nur in `dashboard.js` für den Optimierungs-Tab). Admin-Personas-Karte ruft nur `loadAdminPersonas()` auf.

### Widget-Link-Fix ✅

`moo-bot.js` öffnete `/dashboard.html` (statische Datei in `public/`, bypassed Router). Geändert auf `/dashboard` → Router-Redirect → localStorage-Seite.

Commit: `1723df3`

### Überblick-Seite verbessert ✅

- **Opener als task_image nicht zählen**: `message_count` in `stores/dashboard.js` zählt jetzt nur `role='user' AND content_type != 'task_image'`. Schüler die nur das Aufgabenbild hochgeladen haben, gelten als „noch nicht gechattet".
- **Zusammenfassung kompakter**: Prompt fordert ≤ 4 Stichpunkte (`• ...`) statt gegliederten Fließtext.
- **Neue Karte „Haben gechattet"**: Grüne Chips mit Namen + Zähler, erscheint nach dem Zusammenfassen über der gelben „Noch nicht gechattet"-Karte. API-Response enthält jetzt `studentsPresent[]`.

Commits: `92ec164`, `1cb7af9`, `8eead65`
